PSQL demo

Project Overview

A simple demo made with PSQL, PHP, Python and JQuery. The demo was coded with VSCode and hosted on a local server with XAMPP. PgAdmin4 was used for database management. On the (XAMPP-hosted) webpage the user can sort results by various attributes, either in ascending or descending order. The user can also choose up to three genres to filter the movies, and use the textbox to query movie titles.

Dataset by Rakkesh Araving G from Kaggle. The set included more than 5000 movies, and the following attributes are saved in the database: title, year, IMDB-rating, runtime (min), main genre, side genres, director(s), and actors.

After selecting the the dataset, I did some quick magic with Python - I was still very much in Python mode after my last project. I checked the encoding, skipped bad lines, and created a temporary table with the columns I wanted from the original dataset. Then I picked all unique genres and people and saved them to their own csv-files. With people, the assumption was that a person always has a surname, and that is the last name of the name string - if they only go by one name, that is saved as a surname and first name is set to null. All three csv-files - temp, genres, and personnel - were imported to PgAdmin4, where the proper work began. For example, I used the temp-table to compare names in the personnel-table to create join tables for directors and actors.
